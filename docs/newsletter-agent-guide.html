2 Automated Schedule

Two Windows Task Scheduler tasks keep the agent running automatically. Both are registered under your user account and require no manual action once set up.

📅 Daily Subscriber Sync

Task Name: SoCal AI Newsletter - Daily Sync

Runs: Every day at 8:00 AM

Script: run_sync.bat

Log: data/logs/sync.log

Pulls new subscriber form submissions from Netlify and updates the local subscribers.json cache. Runs one hour before the newsletter send so the list is always current.

📰 Weekly Newsletter Send

Task Name: SoCal AI Newsletter

Runs: Every Tuesday at 9:00 AM

Script: run_newsletter.bat

Log: data/logs/newsletter.log

Runs the full pipeline: subscriber sync → AI research → newsletter writing → email delivery to all active subscribers via Resend.

⚠️
The scheduled tasks only run when your PC is on and logged in. If the machine is off or sleeping on Tuesday at 9 AM, the newsletter will not send automatically. You can trigger a manual run anytime — see Section 4.

3 How Subscribers Are Managed

Subscribe flow

When someone fills out the newsletter sign-up form on the SoCal AI Solutions website, Netlify captures the submission. The daily sync task (8 AM) calls the Netlify Forms API and pulls all new submissions into the local cache file at data/subscribers.json.

Unsubscribe flow

If someone submits the unsubscribe form, their email is added to a separate Netlify form. During every sync, the agent fetches unsubscribe submissions and removes matching emails from the active list before saving. This happens automatically — no manual action needed.

Deduplication

The merge logic deduplicates by email address (case-insensitive). Submitting the same email twice results in only one subscriber record.

Local cache

The file data/subscribers.json is the source of truth for who receives each newsletter. It is updated by every sync and newsletter run. Example entry:

{
  "email": "user@company.com",
  "name": "Jane Smith",         // optional — captured from form
  "subscribed_at": "2026-03-27", // ISO date of first submission
  "source": "netlify"           // "netlify" or "manual"
}

4 Commands Reference

💡
All commands below must be run from the agent's root folder:
cd C:\Users\luisn\Desktop\newsletter-agent

Subscriber management

CommandWhat it doesCost
python main.py --sync Pull latest subscribers from Netlify and update local cache. Shows a list of all active subscribers. Free — no API calls
python main.py --add-sub Manually add a subscriber to the local cache (prompts for email and name). Useful for adding test addresses or VIP contacts without a form submission. Free — no API calls

Newsletter runs

CommandWhat it doesCost
python main.py --dry-run Full pipeline (sync → research → write) but does not send. Saves the finished newsletter HTML to data/last_newsletter.html for review. Use this to preview what would be sent. Claude API tokens only
python main.py --test Full pipeline + sends to TEST_EMAIL only (defined in .env). Use this to verify the full flow — research, writing, and delivery — before a real send. Claude API + Resend (1 email)
python main.py Full pipeline + sends to all active subscribers. This is what the Tuesday scheduled task runs automatically. Claude API + Resend (all subscribers)

Scheduled task management

CommandWhat it doesRequires Admin
python setup_schedule.py Register the weekly Tuesday newsletter task in Windows Task Scheduler. Yes
python setup_schedule.py --add-sync Register the daily 8 AM subscriber sync task. Yes
python setup_schedule.py --status Show the current status of the newsletter task (last run, next run, result code). No
python setup_schedule.py --run Trigger the newsletter task immediately via Task Scheduler (same environment as an automated run). No
python setup_schedule.py --remove Remove the weekly newsletter task from Task Scheduler. Yes
python setup_schedule.py --remove-sync Remove the daily sync task from Task Scheduler. Yes
✅
Recommended test sequence for a new setup:
1. python main.py --sync — verify subscribers are being pulled
2. python main.py --dry-run — check the newsletter quality, open data/last_newsletter.html in a browser
3. python main.py --test — send to yourself and confirm email delivery
4. Let the Tuesday schedule take over

7 Reading the Logs

Each automated run appends output to a log file. Open them in any text editor, or tail them live in PowerShell.

View the latest newsletter log

# Show last 100 lines
Get-Content data\logs\newsletter.log -Tail 100

# Live tail (updates as new lines arrive)
Get-Content data\logs\newsletter.log -Wait

View the latest sync log

Get-Content data\logs\sync.log -Tail 50

8 Troubleshooting

SymptomLikely CauseFix
subscribers.json not updating Daily sync task not registered, or PC was off at 8 AM Run python main.py --sync manually. Re-register with setup_schedule.py --add-sync (Admin).
Fetched 0 from Netlify Forms NETLIFY_FORM_ID is wrong or token expired Check Netlify dashboard → Forms → click your form → copy ID from URL. Update .env.
Newsletter not sending on Tuesday PC was off / sleeping, or task not registered Run python setup_schedule.py --status to confirm task exists. Run python main.py to send manually.
ERROR: Access is denied (schtasks) setup_schedule.py needs Administrator privileges Open Terminal as Administrator: Win+X → Terminal (Admin)
Missing required environment variables .env file missing or incomplete Copy .env.example to .env and fill in all required values. See Section 6.
Emails going to spam Domain not verified in Resend Add and verify the SPF/DKIM DNS records in your domain provider per Resend's setup guide.
Rate limit error from Claude API Research and writing steps hit per-minute token limit The agent auto-pauses 65 seconds between research and writing. If errors persist, try running during off-peak hours.
